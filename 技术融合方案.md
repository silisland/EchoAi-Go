# RV1106边缘龙虾助手 - 技术融合方案

## 项目架构：EchoAI + PicoClaw 技术融合

### 核心策略
在EchoAI成熟的语音/视觉AI技术栈基础上，真正集成PicoClaw的Go语言Agent框架，实现"双方案架构"（云端API + 本地Agent）。

### 技术架构对比

| 层面 | EchoAI原生方案 | PicoClaw集成方案 | 融合方案 |
|------|----------------|------------------|----------|
| **AI对话** | 阿里云DashScope API | PicoClaw本地Agent | 双方案：云端API保底 + PicoClaw增强 |
| **语音识别** | SenseVoice本地模型 | PicoClaw语音插件 | 复用EchoAI的SenseVoice |
| **语音合成** | CosyVoice云端API | PicoClaw TTS | 复用EchoAI的CosyVoice |
| **物体检测** | YOLOv5 + RKNN | 无 | 保留EchoAI视觉能力 |
| **GUI** | LVGL v9.2 | 无显示 | EchoAI LVGL界面展示PicoClaw状态 |
| **通信** | WebSocket + JSON | 飞书/QQ/Telegram Bot | 双通道：本地WebSocket + 远程Bot |
| **内存占用** | ~100MB | ~10MB | 优化后~150MB（RV1106有256MB） |

### PicoClaw在RV1106上的适配挑战

1. **Go语言交叉编译**
   - RV1106是ARM Cortex A7，需要交叉编译Go程序
   - PicoClaw依赖SQLite等CGO库，需要静态链接

2. **资源限制**
   - RV1106只有256MB RAM，需要裁剪PicoClaw功能
   - 去掉非必要插件，保留核心Agent能力

3. **Skill生态适配**
   - PicoClaw的Skill大多面向服务器环境
   - 需要为RV1106开发嵌入式专用Skill（GPIO控制、传感器读取等）

### 融合技术栈

```
硬件层：RV1106 (Cortex A7 + 1TOPS NPU)
    ↓
系统层：Buildroot Linux + PicoClaw (Go二进制)
    ↓
AI层：
    ├── SenseVoice (本地ASR)
    ├── CosyVoice (云端TTS)
    ├── YOLOv5 (本地物体检测)
    └── PicoClaw Agent (本地对话决策)
    ↓
应用层：LVGL GUI + 飞书/QQ Bot
```

### "养龙虾"概念包装

- **虚拟宠物形象**：LVGL界面显示一个可爱的AI龙虾助手
- **养成系统**：通过对话互动、完成任务提升龙虾"成长值"
- **技能学习**：PicoClaw的Skill作为龙虾的"技能"
- **多模态交互**：语音对话 + 表情动画 + 物体识别反馈

**项目简介（技术融合版）：**

> 该项目是一款基于RV1106芯片的嵌入式AI硬件产品，融合开源EchoAI技术栈与PicoClaw轻量化智能体框架。项目采用Cortex A7处理器配合1TOPS NPU，实现语音识别、物体检测和大语言模型交互等核心AI功能。软件层面整合LVGL图形框架、Buildroot Linux系统和Go语言PicoClaw Agent，通过WebSocket协议实现前后端通信，支持双方案AI服务（云端API保底+本地PicoClaw增强）。该机器人能够进行智能对话、环境感知和人机交互，以"AI桌面宠物"形式提供智能家居控制、教育辅助和陪伴服务，实现"养龙虾"的边缘AI智能体体验。

### 技术实现优先级

**Phase 1：EchoAI复刻（2个月）**
1. 搭建RV1106开发环境
2. 复刻DeskBot GUI系统
3. 集成SenseVoice/CosyVoice语音对话
4. 跑通YOLOv5物体检测

**Phase 2：PicoClaw集成（2个月）**
1. 交叉编译PicoClaw for RV1106
2. 开发PicoClaw与EchoAI的通信接口
3. 实现双方案切换逻辑
4. 开发LVGL界面展示PicoClaw状态

**Phase 3：优化包装（1个月）**
1. 内存优化（目标<200MB）
2. 开发"养龙虾"UI动画
3. 集成飞书/QQ Bot远程控制
4. 完善文档和演示

### 技术风险

| 风险 | 等级 | 应对措施 |
|------|------|----------|

| Skill生态不兼容 | 中 | 自主开发嵌入式专用Skill |
| 双方案切换延迟 | 低 | 本地优先，云端兜底 |
