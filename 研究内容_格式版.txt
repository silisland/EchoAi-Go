

（一）项目研究背景

1、国内外研究现状及研究意义

当前，开源 AI 智能体框架在学术界和产业界快速发展，轻量化、边缘化的 AI 部署成为重要趋势。然而现有方案普遍存在资源占用高、依赖云端服务等问题，难以在资源受限的嵌入式设备上稳定运行。

与此同时，国产边缘 AI 芯片技术不断成熟，为端侧 AI 应用提供了新的硬件基础。基于边缘 AI 芯片构建多模态智能交互系统，实现语音识别、物体检测、智能对话等能力的融合，具有重要的技术探索价值和应用前景。

本项目基于国产 RV1106 边缘 AI 芯片，融合成熟的开源技术方案与轻量化智能体框架，探索边缘端轻量化 AI 系统的可行性与优化方法，为边缘 AI 设备的普及应用提供实践参考。

2、项目已有的基础

掌握 C/C++、Python 等编程语言，熟悉嵌入式 Linux、传感器、电路原理等专业知识
熟练使用 Altium Designer、Keil、Proteus、MATLAB 等开发软件
已完成相关开源技术栈的研究与验证，掌握部署方法与技术原理

3、已取得的成绩

能够独立设计程序对嵌入式系统进行控制
了解了语音识别算法和相关 AI 模型的工作原理
已在开发环境成功验证基础 AI 功能，已有初步的方案和原理图

4、已具备的条件

具备嵌入式开发环境（Keil、VS Code、Ubuntu WSL2）、仿真工具、硬件设计软件等开发工具
具备相关开源项目代码库和技术文档
具备相关编程语言基础和硬件开发经验

5、尚缺少的条件及方法

需要在 RV1106 平台上完成智能体框架的首次部署适配
需要对智能体 Skill 体系进行嵌入式场景裁剪优化
需要设计语音交互与智能体执行的闭环控制

（二）项目研究目标及主要内容

1、研究目标

预期在 RV1106 平台上实现开源技术栈与轻量化智能体框架的融合，构建"云端 API + 本地双方案"架构，支持语音对话、物体检测、GUI 交互和智能任务执行。

该装置可作为 AI 桌面助手，用于学习桌面、智能家居控制、教育辅助和陪伴服务等场景。

2、主要研究内容

本 AI 桌面助手由 RV1106 核心板、触摸屏、音频模块、摄像头和 WiFi 模块组成闭环控制系统。系统总体构成包括：

（1）双方案 AI 服务：采用云端 API 与本地智能体相结合的架构
本地智能体层：基于开源方案实现语音识别、物体检测、图形界面
智能体框架层：轻量化 Agent 框架，内存占用约 10MB，适合资源受限设备
融合层：设计通信接口，实现语音输入、AI 处理、智能执行、GUI 反馈的完整闭环

（2）嵌入式 Linux 控制：基于 Buildroot 定制系统，集成音频驱动、视频驱动、显示驱动

（3）触摸屏显示：采用 LVGL 图形框架，实现主界面、语音对话界面、物体检测画面等功能页面

（4）音频采集播放：通过 PortAudio 实现麦克风音频采集，经 Opus 编码后通过 WebSocket 传输

（三）项目创新特色概述

1. 技术融合创新：首次将成熟的开源技术栈与轻量化智能体框架深度融合，实现多模态 AI 桌面助手

2. 双方案架构设计：采用云端 API 保底 + 本地智能体增强的双方案架构，兼顾稳定性与隐私性

3. 国产芯片突破：在 RV1106 国产芯片上实现完整 AI 技术栈与智能体框架的部署融合

4. 交互体验创新：以 AI 桌面助手形式支持语音、视觉、触控多模态交互

（四）项目研究技术路线

1. 软硬件结合、实践与仿真结合：使用 Keil/VS Code 配合 Ubuntu 进行嵌入式编程，使用 Proteus 进行仿真调试

2. 方案规划与架构选择：在成本可控的前提下为系统选择最优架构，采用双方案架构平衡云端与本地的优势

3. 云端 API 方案：系统可通过云端 API 调用快速实现语音识别、大语言模型对话和语音合成功能，语音服务由服务端提供，系统负责音频采集和播放

4. 本地智能体方案：采用轻量化智能体框架，内存占用仅约 10MB，单一静态二进制无运行时依赖，适合资源受限的嵌入式设备，可离线运行核心功能
