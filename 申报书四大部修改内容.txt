二、项目方案
============================================================

（一）项目研究背景

1、国内外研究现状及研究意义

2026年初，开源AI智能体框架OpenClaw席卷全球，"养龙虾"成为科技圈最热话题。然而OpenClaw需至少4GB内存，PicoClaw应运而生——以10MB内存占用将Agent能力下沉到几乎任何Linux设备。国内RV1106等国产芯片推动端侧AI普及，AI智能体边缘化成为重要发展趋势。

与此同时，开源社区涌现出EchoAI等成熟的嵌入式AI方案，其基于RV1106芯片实现了SenseVoice语音识别、YOLOv5物体检测、LVGL图形界面等完整功能，为嵌入式AI硬件开发提供了可靠的技术参考。本项目在EchoAI开源技术栈基础上，融入PicoClaw轻量化智能体框架，探索边缘端"AI桌面宠物"的创新形态，为国产嵌入式平台与开源AI生态融合提供实践参考。

OpenClaw于2025年底开源，4个月内GitHub星标突破25万；PicoClaw由Sipeed团队用Go语言重写，内存仅约10MB，支持多架构；EchoAI项目在RV1106上实现了完整的语音/视觉AI功能。三者结合，有望打造兼具技术成熟度与创新性的边缘AI产品。

2、项目已有的基础

掌握C/C++、Python、Go语言，熟悉嵌入式Linux、传感器、电路原理等专业知识；熟练使用Altium Designer、Keil、Proteus、MATLAB等开发软件；已完成EchoAI开源项目复刻，掌握SenseVoice/YOLOv5/LVGL技术栈；已完成PicoClaw架构研究，理解Agent系统设计原理。

3、已取得的成绩

能够独立设计程序对嵌入式系统进行控制，已在WSL2平台成功复刻EchoAI项目并验证语音对话、物体检测、GUI显示功能，深入理解其WebSocket通信、RKNN推理等核心技术；已有初步的方案和原理图，现已投入开发。

4、已具备的条件

具备嵌入式开发环境（Keil、VS Code、Ubuntu WSL2）、仿真工具、硬件设计软件（Altium Designer）等开发工具；具备EchoAI开源项目代码库和PicoClaw源码；具备相关编程语言基础和硬件开发经验。

5、尚缺少的条件及方法

需要在RV1106平台上完成PicoClaw与EchoAI技术栈的融合适配，需要对Claw Skill体系进行嵌入式场景裁剪优化，需要设计语音交互与Agent执行的闭环控制。

（二）项目研究目标及主要内容

1、研究目标

预期在RV1106平台上实现EchoAI技术栈与PicoClaw智能体框架的融合，构建"云端API + 本地双方案"架构，支持语音对话、物体检测、GUI交互和Agent任务执行。技术突破点包括：
- PicoClaw在RV1106上的首次部署适配
- EchoAI技术栈（SenseVoice/YOLOv5/LVGL）与PicoClaw的深度集成
- 语音交互与Agent执行的闭环设计
- 双方案AI服务的智能切换机制

该装置可作为"AI桌面宠物"，用于学习桌面、智能家居控制、教育辅助和陪伴服务等场景。

2、主要研究内容

本AI桌面助手由RV1106核心板、触摸屏、音频模块、摄像头和WiFi模块组成闭环控制系统。系统总体构成包括：

（1）双方案AI服务：采用云端API与本地PicoClaw+EchoAI相结合的架构。
- EchoAI层：基于开源项目实现SenseVoice语音识别、YOLOv5物体检测、LVGL图形界面
- PicoClaw层：Go语言编写的轻量化Agent框架，内存占用约10MB，单一静态二进制无运行时依赖
- 融合层：设计EchoAI与PicoClaw的通信接口，实现语音输入→AI处理→Agent执行→GUI反馈的完整闭环

（2）嵌入式Linux控制：基于Buildroot定制系统，集成音频驱动（PortAudio）、视频驱动（RKNN）、显示驱动（Framebuffer）。

（3）触摸屏显示：采用LVGL v9.2图形框架，实现主界面、语音对话界面、物体检测画面、Agent状态显示等功能页面。

（4）音频采集播放：通过PortAudio实现麦克风音频采集（16kHz单声道），经Opus编码后通过WebSocket传输；接收服务器返回的音频流，解码后播放。

（三）项目创新特色概述

1. 技术融合创新：首次将开源EchoAI技术栈（SenseVoice/YOLOv5/LVGL）与PicoClaw轻量化Agent框架深度融合，实现"能看、能听、能思考、能执行"的多模态AI桌面宠物。

2. 双方案架构设计：采用"云端API保底 + 本地PicoClaw+EchoAI增强"的双方案架构，既保证云端方案的稳定性和功能丰富性，又具备本地方案的隐私性、离线可用性和低延迟响应。

3. 国产芯片突破：在RV1106国产芯片（256MB RAM）上首次实现PicoClaw与完整AI技术栈的部署，验证边缘端轻量化AI智能体的可行性。

4. "养龙虾"概念落地：以"AI桌面宠物"形式包装技术实现，支持语音交互、物体识别反馈、Agent任务执行，打造"动嘴养龙虾"的创新用户体验。

（四）项目研究技术路线

1. 软硬件结合、实践与仿真结合：使用Keil/VS Code配合Ubuntu（WSL2/虚拟机）进行嵌入式编程，使用Proteus进行仿真调试。使用Python进行EchoAI服务端开发，使用Go语言进行PicoClaw集成，使用MATLAB进行参数调优。

2. 方案规划与架构选择：在成本可控的前提下为系统选择最优架构。采用双方案架构：
- 云端方案：阿里云DashScope API提供语音识别（SenseVoice）、大语言模型对话（Tongyi Qianwen）、语音合成（CosyVoice）
- 本地方案：PicoClaw Agent + EchoAI技术栈（SenseVoice本地模型 + YOLOv5 RKNN推理 + LVGL GUI）

3. EchoAI技术实现：
- 语音识别：集成SenseVoice开源模型，实现本地ASR
- 物体检测：YOLOv5模型转换为RKNN格式，在NPU上加速推理
- 图形界面：LVGL v9.2实现多页面导航系统
- 通信协议：WebSocket + JSON + Opus音频流传输

4. PicoClaw集成方案：
- 交叉编译PicoClaw for ARM Cortex A7
- 开发EchoAI-PicoClaw通信接口
- 裁剪Claw Skill体系，保留核心Agent能力
- 设计Agent状态显示UI（LVGL界面展示"龙虾"状态）

三、项目创新特色与实施方案
============================================================

（根据原申报书格式，此处内容为"项目创新特色概述"的扩展版）

本项目在技术创新和产品形态上具有以下特色：

1. 开源技术栈融合创新
不同于单一依赖云端API或单一本地方案，本项目深度融合EchoAI（GitHub: No-Chicken/Demo4Echo）成熟开源方案与PicoClaw轻量化Agent框架。EchoAI提供经过验证的SenseVoice语音识别、YOLOv5物体检测、LVGL图形界面技术；PicoClaw提供Go语言编写的轻量级Agent执行能力。两者结合，实现"能看、能听、能思考、能执行"的完整AI能力闭环。

2. 双方案智能切换架构
设计智能切换机制：网络良好时优先使用本地PicoClaw+EchoAI方案（隐私保护、低延迟）；网络受限或需要复杂对话时切换至云端API（功能丰富、稳定性高）。通过WebSocket协议实现双方案的无缝切换和状态同步。

3. 边缘AI完整技术链
从音频采集（PortAudio）→语音识别（SenseVoice）→AI对话（PicoClaw Agent）→语音合成（CosyVoice）→音频播放，构建端到端的边缘AI处理流水线。物体检测采用YOLOv5+RKNN方案，在1TOPS NPU上实现实时推理。

4. "AI桌面宠物"产品形态
以"养龙虾"概念包装技术实现，设计可爱的AI助手形象（LVGL动画），支持：
- 语音唤醒和对话
- 摄像头识别桌面物品并反馈
- 执行Agent任务（如控制智能家居）
- 养成系统：通过对话互动提升"亲密度"
